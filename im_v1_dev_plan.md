# 企业 IM 最小化生产落地计划（V1.0 正式版）

## 核心理念

**只做能真正用起来的功能，砍掉一切非必需的"体验"和"扩展"。**

目标：3-4 周内完成首个可部署、可使用的正式版本，投入实际办公场景。

## V1.0 明确砍掉的功能（后续版本再议）

- 消息撤回、消息回复、@提及
- 表情包、消息已读回执
- 会话免打扰、聊天记录搜索
- 角色权限分级（V1 只做管理员/普通用户两档）
- 操作日志、登录日志
- 敏感词过滤、消息审计
- 企业公告、组织架构导入
- Electron 桌面打包（V1 用浏览器访问，局域网部署）
- MinIO（V1 用本地磁盘存储文件）
- 多端同步、系统托盘通知
- 移动端

## 技术栈精简

| 层级 | V1 选型 | 说明 |
|---|---|---|
| 后端 | Java 21 + Spring Boot 3.x | 单体应用，包内模块拆分 |
| 安全 | Spring Security + JWT | 无状态认证 |
| 数据库 | MySQL 8.0 | 单库，不需要读写分离 |
| 缓存 | Redis | 仅存 Token 和在线状态 |
| 实时通信 | Spring WebSocket | 原生方案，不引入额外组件 |
| 文件存储 | 本地磁盘 | 项目 static/upload 目录 |
| ORM | MyBatis-Plus | 减少手写 SQL |
| 前端客户端 | Vue 3 + TypeScript + Vite | 浏览器访问 |
| 管理后台 | Vue 3 + Element Plus | 浏览器访问 |
| 反向代理 | Nginx | 统一入口 |
| 部署 | Docker Compose | 一条命令启动全部服务 |

## 数据库核心表（只建 6 张）

```text
sys_user       用户表
sys_dept       部门表

im_conversation         会话表
im_conversation_member  会话成员表
im_message              消息表
im_file                 文件表
```

（角色权限用 sys_user 的 role 字段区分 admin/user，不需要 RBAC 三张表）

---

## 关键原则

1. **不做过度设计** — V1 不需要微服务、不需要消息队列、不需要 ES
2. **不追求完美 UI** — 能看、能用、不卡，就是合格
3. **先跑通再优化** — 功能闭环优先于代码优雅
4. **每天有产出** — 以天为单位规划，不堆积到周末
5. **服务端不存文件以外的东西** — 不引入 MinIO，减少部署依赖
